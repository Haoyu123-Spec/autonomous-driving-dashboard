# 基于 Dueling DQN 的多 AGV 智能决策与碰撞避免 —— 技术呈现报告

---

## 1. 问题定义与应用背景

在智能仓储、港口物流、柔性制造等场景中，多台 AGV（自动导引车）需要在共享空间中独立导航至各自目标点，同时避免相互碰撞及与动态障碍物的碰撞。

**核心挑战**：
- 多智能体同时决策，动作空间随 AGV 数量指数增长
- 部分可观测性：每台 AGV 仅能感知自身周围局部信息
- 动态环境：障碍物随机运动，紧急订单随机插入
- 稀疏奖励：到达目标才能获得正向激励，探索效率极低

**形式化定义**：该问题建模为去中心化的部分可观测马尔可夫决策过程（Dec-POMDP）。每个智能体独立观测环境并输出动作，共享同一奖励信号。智能体之间不做显式通信——协同行为通过奖励函数隐式学习。

## 3. Dueling DQN 网络设计

### 3.1 基本原理

标准 DQN 直接输出 Q(s, a)。Dueling DQN 将 Q 值分解为两部分：

$$Q(s, a) = V(s) + \left(A(s, a) - \frac{1}{|A|}\sum_{a'}A(s, a')\right)$$

- **V(s)**（状态价值）：当前位置有多"好"
- **A(s, a)**（优势函数）：某个动作比平均好多少

这一分解的关键优势：在很多状态下，具体选择哪个动作并不重要（例如空旷区域直行即可），此时 V(s) 已经能准确评估状态好坏，A(s, a) 只需学习微小差异。相比标准 DQN 必须学习每个动作的精确 Q 值，Dueling 架构的样本效率更高。

### 3.2 网络结构（`network.py:59-120`）

```
输入: state_dim (自适应维度)
  │
  ├─ Feature 层: Linear(state_dim, 256) → ReLU
  │
  ├─ [可选] Self-Attention 分支:
  │    state → 拆分为 (n_entities, 2) → Linear(2, 256)
  │         → MultiheadAttention(h=4) → LayerNorm → Mean Pool
  │         → 与 Feature 输出拼接 → Linear(512, 256) → ReLU
  │
  ├─ 无 Attention: Linear(256, 256) → ReLU
  │
  ├─ Value 流: Linear(256, 128) → ReLU → Linear(128, 1)
  │
  └─ Advantage 流: Linear(256, 128) → ReLU → Linear(128, 5)
       │
       └─ 合并: Q = V + A - mean(A)
```

### 3.3 Self-Attention 模块（创新点）

AGV 的观测由多组二维向量组成（自身位置、目标位置、其他 AGV 相对位置、障碍物相对位置）。这些向量之间天然存在"实体交互"语义——其他 AGV 靠近时需要避让、障碍物逼近时需要转向。

Self-Attention 模块将观测按每 2 维拆分为实体序列，通过多头注意力学习跨实体交互：

- 将 state 向量 reshape 为 `(B, n_entities, 2)` 的三维张量
- 每个 2D 实体通过投影层映射到 256 维空间
- 4 头注意力在实体间传递信息
- 残差连接 + LayerNorm 确保训练稳定
- 均值池化后再与原始特征拼接

这相当于让网络"注意"最危险的邻近 AGV 或障碍物，而不是简单依赖排序后的固定位置编码。

### 3.4 NoisyNet 探索（`network.py:8-57`）

传统 ε-greedy 探索在所有状态下均匀随机，效率低下。NoisyNet 在权重上施加可学习的高斯噪声：

```python
weight = weight_mu + weight_sigma * ε    # ε ~ N(0, 1)
```

关键的因子化噪声设计使用了 `sign(ε) × sqrt(|ε|)` 变换，确保噪声有界且稳定。噪声的 σ 参数通过梯度下降学习——网络自行决定哪些层需要更多探索。**实际训练中，NoisyNet 的 σ 噪声在复杂场景下会过大导致 μ 权重崩溃（NaN），因此最终配置关闭了 NoisyNet，回退到 ε-greedy（ε 从 1.0 指数衰减至 0.02）。**

## 4. 复合奖励函数设计

稀疏奖励（仅到达目标给分）在多 AGV 场景中几乎无法学习。本文设计了五元复合奖励函数：

### 4.1 方向引导奖励 R_direction

$$R_{direction} = \log(\max(d, 1)) \cdot \exp(-\alpha \cdot \theta)$$

| 符号 | 含义 |
|------|------|
| d | AGV 到目标的欧氏距离 |
| θ | 当前航向与目标方向的夹角（弧度） |
| α = 1.5 | 角度惩罚系数 |

**设计考量**：
- `log(d)` 项：距离远时奖励信号较强（鼓励靠近），距离近时自然减弱，避免在目标附近过度激励
- `exp(-αθ)` 项：偏离目标方向时指数级衰减，引导 AGV 保持朝向目标
- 这一奖励只在航向非零时生效（即 AGV 执行了移动动作），停留（action=0）不发

### 4.2 碰撞惩罚 R_collision

| 碰撞类型 | 惩罚值 | 处理方式 |
|----------|--------|----------|
| 墙壁/边界碰撞 | -10 | AGV 立即终止 |
| 动态障碍物碰撞 | -10 | AGV 立即终止 |
| AGV 间碰撞 | -20 | 双方均终止 |

AGV 间碰撞惩罚（-20）大于墙壁碰撞（-10），因为 AGV 间碰撞在真实场景中的代价更高（设备损坏、任务中断），且相比撞墙更难预测。

### 4.3 步进惩罚 R_step = -1

每步固定 -1 惩罚，驱动力来自：AGV 必须以最短路径完成任务，否则累积的步数惩罚会抵消到达奖励。这与路径规划中的"最短路径偏好"一致。

### 4.4 三区协同奖励 R_coop（创新点）

这是多 AGV 场景的核心设计：

| 区域 | 距离范围 | 奖励/惩罚 |
|------|----------|-----------|
| 危险区 | d < 2.0 | -5（需要紧急避让） |
| 协同区 | 2.0 ≤ d ≤ 4.0 | +1（适度靠近，协同共存） |
| 安全区 | d > 4.0 | 0（不干扰） |

设计目标：
- **危险区负奖励**教会 AGV 保持安全距离，防止"贴脸"碰撞
- **协同区正奖励**鼓励 AGV 在不碰撞的前提下适度靠近——这与纯避障不同，后者会让 AGV 过度保守。适度的空间共享提升了整体吞吐量
- 三区制避免了"全或无"的二元惩罚，提供了连续梯度

### 4.5 到达奖励 R_complete = +100

到达目标点的一次性大额奖励。紧急订单额外附加 +50 bonus。

## 5. 训练策略

### 5.1 Double DQN（`agent.py:87-90`）

使用在线网络选择最优动作，目标网络评估价值：

```python
next_a = online(ns).argmax(dim=1)    # 在线网络选择
target_q = target(ns)[next_a]         # 目标网络评估
```

这解耦了动作选择与价值评估，显著缓解 Q 值高估偏差——在碰撞场景中，高估某个方向的安全性可能导致灾难性决策，Double DQN 让这种偏差大幅降低。

### 5.2 优先经验回放 PER（`network.py:129-178`）

均匀采样会浪费大量时间在"简单"样本上（空旷区域直行）。PER 按 TD-error 的 α 次方加权采样：

$$P(i) \propto |\delta_i|^\alpha$$

同时用重要性采样权重校正偏差：

$$w_i = (N \cdot P(i))^{-\beta}$$

β 从 0.4 线性增长至 1.0，逐步从"偏向高 TD-error 样本"过渡到"无偏估计"。**实测效果：高碰撞和成功到达的样本被重复学习，加速了关键经验的消化。**

### 5.3 课程学习 Curriculum Learning（`train.py:64-83`）

从简单场景逐步过渡到复杂场景，6 个阶段共计 3000 episode：

| 阶段 | Episode | AGV | 动态障碍 | 紧急订单 | 目标 |
|------|---------|-----|----------|----------|------|
| S1 | 1-400 | 2 | 0 | 关闭 | 学会单路径规划 |
| S2 | 401-800 | 3 | 1 | 关闭 | 引入简单避障 |
| S3 | 801-1300 | 3 | 2 | 关闭 | 多障碍协同 |
| S4 | 1301-1800 | 4 | 2 | 关闭 | 多 AGV 协同避碰 |
| S5 | 1801-2400 | 4 | 3 | 开启 | 引入紧急订单 |
| S6 | 2401-3000 | 5 | 4 | 开启 | 全复杂度压测 |

课程学习的核心思想：**如果一开始就在 5 AGV + 4 障碍的复杂场景中学习，agent 几乎无法遇到成功到达的样本（碰撞率为 100%），导致无从学起。**从简单场景开始积累了成功路径的经验后，再逐步增加难度，网络有了正样本基础，能够学到真正的避碰策略而非随机探索。

### 5.4 硬更新 vs 软更新（`agent.py:103-111`）

- **硬更新**（`target_update_freq=100`）：每 100 步完整复制在线网络到目标网络，更新彻底但可能引入波动
- **软更新**（Polyak averaging，tau=0.005）：每步微量混合，更新平滑但收敛慢

经过实验对比，硬更新在复合奖励场景下更稳定——软更新的慢速追踪在奖励信号突变（碰撞 -> 到达）时跟不上。

### 5.5 紧急订单插入（`env.py:147-172`）

模拟真实物流中的"抢单"场景：
- 每 episode 30% 概率触发
- 在固定步数（10 到 max_steps×40% 之间随机）重定向某 AGV 的目标
- 新目标距离 AGV 在 `0.15×world_size` 到 `0.35×world_size` 之间
- 完成后额外 +50 bonus

紧急订单考验的是 AGV 在目标突变后的**重规划能力**——能否在已有碰撞风险的约束下快速调整路径。

## 6. 观测空间设计

状态向量的维度随课程阶段动态调整，但网络输入使用全局最大维度以保证结构统一：

```
| 索引范围 | 内容 | 维度 |
|----------|------|------|
| 0:2 | 自身位置（归一化到 [0,1]） | 2 |
| 2:4 | 目标位置（归一化） | 2 |
| 4:18 | 最近 7 个其他 AGV 的相对位置 | 14 |
| 18:26 | 最多 4 个动态障碍物的相对位置 | 8 |
| **总计** | **针对 8 个 AGV + 4 障碍的固定维度** | **26** |
```

关键设计：
- 其他 AGV 按距离排序，只保留最近的 `max_agents-1` 个。剔除自身后，远距离 AGV 被截断——降低计算量，且远距离 AGV 决策影响力小
- 所有坐标除以 `world_size` 归一化到 [0, 1]，保证数值稳定

## 7. 实验与结果

### 7.1 训练曲线

最终提交（commit `d49a248`）完成了完整的 3000 集课程学习：

- **S1**（2 AGV, 0 障碍）：avg100 收敛至 **+29.4**——简单场景下几乎无碰撞，AGV 学会直线导航
- **S4**（4 AGV, 2 障碍）：avg100 约 **-19**——随着复杂度提升，碰撞增加，但策略仍在学习
- 整体碰撞率从初期 ~100% 下降到后期间歇性碰撞

### 7.2 奖励曲线分析

从训练指标中观察到：
- **波动模式**：-19（碰撞 episode）↔ +31（成功 episode）交替——说明策略学会了避碰，但不是 100%
- **渐进改善**：成功 episode 的间隔逐渐缩小

### 7.3 面临的挑战

1. **奖励信号矛盾**：R_step(-1) 驱动 AGV 快速行动，但快速行动增加碰撞风险（-20）。AGV 需要在"快"和"安全"之间找到最优平衡
2. **多智能体非平稳性**：每个 AGV 的策略在变，其他 AGV 从它的视角就是"非平稳环境"
3. **稀疏正样本**：复杂场景下碰撞率仍偏高，正样本不足以完全覆盖状态空间

## 8. 从 v1 到 v2 的演进对比

| 维度 | `dueling_dqn_agv.py` (v1) | `rl_advanced/` (v2) |
|------|---------------------------|---------------------|
| 奖励函数 | 到达+30, 碰撞-20, 步进-0.05, 距离势能 | 5 元复合：方向引导 + 三区协同 + 紧急 bonus |
| 网络 | 纯 Duelling DQN | Dueling + Self-Attention + NoisyNet(可选) |
| 经验回放 | 均匀采样 deque | PER 按 TD-error 加权 |
| 训练策略 | 固定 4 AGV 平铺训练 | 6 阶段课程学习 |
| 环境复杂度 | 无动态障碍、无紧急订单 | 动态障碍物 + 紧急订单插入 |
| 目标更新 | 仅软更新 tau=0.005 | 硬更新(N=100) + 软更新可选 |
| 损失函数 | MSE Loss | Smooth L1 Loss (Huber) |
| 梯度裁剪 | 硬编码 10.0 | 可配置 `grad_clip` |

**演进动因**：v1 在简单场景（固定 4 AGV 无障碍）可以学到基本避碰，但在引入动态障碍后完全失效——均匀经验回放中 99% 的样本是"正常行驶"，碰撞样本被淹没，奖励信号不够密集。v2 通过复合奖励和 PER 解决了这两个问题。

## 9. 技术贡献总结

1. **五元复合奖励函数**：将稀疏到达奖励扩展为密集引导信号，方向奖励 + 三区协同 + 碰撞惩罚的组合形成了平滑的奖励景观，解决了"无正样本可学"的根本问题

2. **Self-Attention 实体交互**：将观测向量重新解释为实体序列，用多头注意力建模 AGV-AGV 和 AGV-障碍物之间的交互关系，比固定排序编码更灵活

3. **课程学习递进策略**：6 阶段从 2 AGV + 0 障碍递进到 5 AGV + 4 障碍 + 紧急订单，每个阶段在前一阶段积累的策略基础上微调——这种设计让网络始终有正样本信号

4. **PER + Double DQN 组合**：优先回放确保碰撞和成功样本被充分学习，Double DQN 避免在复杂碰撞场景中高估安全性

5. **紧急订单场景**：模拟真实物流中的动态重规划需求，验证了策略在目标突变时的适应性

---

## 10. 改进方向

1. **Multi-Agent RL 方法**：当前各 AGV 独立决策，可引入 MADDPG/QMIX 等集中训练-分散执行框架，利用全局信息
2. **图神经网络编码**：AGV 间关系天然是图结构（节点=实体，边=距离/关系），GNN 比 Self-Attention + 排序更自然
3. **奖励自适应缩放**：各奖励组件的相对权重目前是手工设定的，可通过元学习或 PopArt 自适应调整
4. **混合探索策略**：ε-greedy 和 NoisyNet 各有局限，可引入基于计数的内在奖励（RND）鼓励访问未探索区域
